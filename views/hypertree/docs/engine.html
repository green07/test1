エンジンは省資源型
大量なデータを高速で処理するプログラムを「エンジン」とするＨＴが持つ多彩な問題解決力は、これまでの技術に比べて画期的なレベルとなっています。

このエンジンの優れた特長は「省資源型」であることです。

  1. コアとなる“階層構造のデータを高速で処理する手続き”は、特許認定を受けた「三分木を扱えるインデックス・テーブル」を駆使したシンプルなもの
  2. 大量データの処理にＳＯＲＴを必要としないので、主メモリ/ディスク上のＷＯＲＫスペースなしで処理ができる（主メモリ/ディスク等の容量が小さいシステムでもＨＴが使える）
  3. ベクトル化という着想を取り入れ、大型のマトリックス計算等で係数の大部分はゼロという場合にゼロの個所の計算をしない（ムダなスペースも取らない）
  4. 処理内容を定義する式の記述に“再帰(recursive)”表現が使えるので、プログラムを短く分かり易く書くことができる
このように、スペース的にも時間的にもムダのなさを極めていることが、結局、「同じ処理結果を出す上での速さ」の源 になっているのです。
